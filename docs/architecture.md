# Архитектура

Преглед на системата на едно място и указател към записаните решения. **Архитектурните решения
живеят като ADR в [`adr/`](adr/README.md)** — този файл е картата, не източникът на истина за
отделните решения. Стекът и базовите конвенции са записани в
[ADR-0001](adr/0001-cloudflare-monorepo-stack.md).

## Поток на данните

```
НКР (nkr.government.bg) ──────┐
Стар НКР (nkrold) ────────────┤   tools/harvest (Python, BG IP, bootstrap)
data.egov.bg API ─────────────┼──▶ + apps/etl (Worker, инкрементален refresh)
МЕ / МТ регистри ─────────────┘        │  raw (R2) → normalize → unify → derive flags → precompute
                                       ▼
                                 D1 (една база на среда)
                                       ▲
                                       │  read-only
              граждани / журналисти ──▶  koncesii (RR v7 SSR Worker)
```

Данните пристигат като **периодични извличания**, не като жива емисия. Регистрите нямат
webhook-ове и режат datacenter IP-та, затова извличането е двустепенно
([ADR-0002](adr/0002-two-stage-ingest.md)):

1. **Bootstrap** — Python инструментите в `tools/harvest/` (пълно обхождане на НКР, стария
   регистър и data.egov.bg) се пускат ръчно от българско IP и качват суровия резултат в R2.
2. **Инкрементален refresh** — `koncesii-etl` Worker-ът тегли новите/променените партиди
   (експортът на НКР е една заявка от ~500 KB) през BG residential proxy или self-hosted
   runner и обновява D1.

ETL-ът пише в D1; explorer-ът чете от **същия** D1. В v1 няма публичен write път и няма
автентикация.

## Двата Worker-а

| Worker         | Роля                                                                 |
| -------------- | -------------------------------------------------------------------- |
| `koncesii`     | SSR explorer (`apps/web`) — чете D1 директно, edge-кешира HTML.       |
| `koncesii-etl` | cron-задействан refresh Workflow (`apps/etl`) — пише в D1.            |

## Слоеве на данните

| Слой        | Къде живее           | Съдържание                                                       |
| ----------- | -------------------- | ----------------------------------------------------------------- |
| raw         | R2                   | сурови HTML/TSV/JSON от източниците, immutable, с дата на снемане |
| staging     | D1 `raw_*` таблици   | парснати записи 1:1 с източника, с `source` префикс               |
| domain      | D1 domain таблици    | единният модел: concessions / grantors / concessionaires / …      |
| derived     | D1 `flag_*`, rollups | изчислените индикатори и precompute агрегати                      |

Единният модел и dedup правилата между източниците (една концесия често съществува и в НКР,
и в общинския регистър на data.egov.bg) — [`core-scope.md`](core-scope.md) и
[ADR-0003](adr/0003-nkr-wins-dedup.md).

## Рендиране и сигурност (резюме)

Пре-рендиране за статичните страници, SSR + edge кеш за
детайлните/explorer страници, клиентски „острови" за интерактивното (картата е най-големият
остров — MapLibre GL, [ADR-0004](adr/0004-map-rendering.md)). Приоритет интегритет ≈
наличност ≫ конфиденциалност (данните са публични). Read пътят е read-only; строг CSP;
D1 само с bound параметри.

## Индикаторите за риск

Флаговете са детерминистични, изчисляват се в ETL derive стъпката и имат публична методология
в [`red-flags.md`](red-flags.md). Никога не се изчисляват в заявка на живо — потребителят
вижда точно това, което методологията описва, върху точно тази версия на данните.
