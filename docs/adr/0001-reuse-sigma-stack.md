# ADR-0001 — Преизползване на стека и конвенциите на СИГМА

- **Статус:** Прието
- **Дата:** 2026-07-10
- **Обхват:** цялото хранилище

## Контекст

КОНЦЕСИИ е сестрински проект на СИГМА: същият тип платформа (read-only прозрачност върху
държавен регистър), същата целева публика, същият cross-link по ЕИК, същият екип и същите
AI-агентни работни потоци. Обемът данни е с два порядъка по-малък (хиляди партиди срещу
стотици хиляди договори). Алтернативата — самостоен стек (напр. Laravel + Postgres, който
екипът също владее) — би дала позната среда, но два кодови стила, две деплой истории и нулево
преизползване на вече решените проблеми (CSP, достъпност, integrity gate, blue/green D1).

## Решение

Копираме СИГМА едно към едно: pnpm + turbo монорепо; Cloudflare Workers (React Router v7 SSR)
+ D1 + R2 + Queues/KV; trunk-based с conventional commits; AGENTS.md/CLAUDE.md механизъм;
същият CI (gitleaks → lint → typecheck → test → docs integrity); ADR практиката; issue/PR
шаблоните. Отклонения от СИГМА се допускат само където домейнът ги налага и се записват като
ADR.

## Последствия

- Плюс: контрибутор на единия проект е контрибутор и на другия без учене; решените в СИГМА
  проблеми (dedup модел, value_flag, rollback) се пренасят наготово; обща инфраструктурна
  сметка.
- Минус: зависимост от Cloudflare екосистемата; D1 лимитите важат (но при този обем не са
  фактор).
- Follow-up: изрично проследяваме upstream промените в конвенциите на СИГМА и ги
  синхронизираме.
