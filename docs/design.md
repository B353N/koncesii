# Дизайн — визуален език на koncesii.com

Обвързващата спецификация за интерфейса (одобрена концепция v1, 10.07.2026). Промени по
дизайна минават като PR към този файл. Мокъпите на четирите ключови страници са
презентирани отделно (артефакт „КОНЦЕСИИ — дизайн концепция v1"); този документ е
източникът на истина за токените и правилата.

## Посока: „регистър на хартия"

Официалната тежест на държавния регистър, поднесена с яснотата на добра журналистика.
Четири принципа:

1. **Регистър, не бюлетин** — плътни таблици, подравнени числа, стабилна навигация;
   данните са героят, не илюстрациите.
2. **Числото носи източник** — до всяка сума/срок стои връзка „произход"
   (компонентът `prov` по-долу). Без изключения.
3. **Флагът е факт** — неутрален език, формулата и входните числа на един ред,
   щриховка вместо аларма (виж „Подписът").
4. **Хартиена яснота** — светла тема (v1), висок контраст, WCAG 2.1 AA, без тежки
   клиентски зависимости извън картата.

## Палитра

Шест цвята, всеки със задача. Червеното и охрата са **резервирани изключително за
индикаторите** — никой друг елемент не ги ползва.

| Токен         | Стойност  | Употреба                                  |
| ------------- | --------- | ----------------------------------------- |
| `--ink`       | `#1B2A2C` | текст, тежки разделители                  |
| `--water`     | `#14464C` | връзки, действия, партидната лента        |
| `--water-br`  | `#0F6A6B` | hover/фокус състояния                     |
| `--paper`     | `#FBFBF8` | фон на страницата                         |
| `--raised`    | `#FFFFFF` | карти, input-и, редове                    |
| `--limestone` | `#E3E4DC` | линии, рамки, тихи разделители            |
| `--stone`     | `#5E6E6C` | вторичен текст; индикатори с ниска тежест |
| `--oxide`     | `#A63A21` | само индикатори с висока тежест           |
| `--ochre`     | `#8F6400` | само индикатори със средна тежест         |

## Типография

| Роля            | Шрифт         | Употреба                                         |
| --------------- | ------------- | ------------------------------------------------ |
| display         | Literata      | заглавия на страници и секции, wordmark          |
| текст/интерфейс | IBM Plex Sans | основен текст, навигация, контроли               |
| данни           | IBM Plex Mono | партидни номера, ЕИК, суми, срокове, флаг кодове |

- И трите са с пълна кирилица и OFL лиценз, **self-hosted** в `apps/web/public/fonts`
  (woff2, subset latin+cyrillic). Външни font CDN-и са забранени (строг CSP).
- Числата в таблици винаги с `font-variant-numeric: tabular-nums`, подравнени вдясно.
- Основен текст ≤ 68ch ширина; заглавия с `text-wrap: balance`.

## Подписът: кадастралната щриховка

Флагнатите сделки се маркират като обременени парцели върху геодезична карта —
**45° диагонална щриховка** в цвета на тежестта. Никога плътен цвят, никога икони на
аларма/предупреждение.

```css
/* висока тежест; аналогично с --ochre и --stone за средна/ниска */
background: repeating-linear-gradient(
  45deg,
  transparent 0 5px,
  rgba(166, 58, 33, 0.28) 5px 6px
);
```

Тежести: висока = oxide (LOW_PAYMENT, SINGLE_BIDDER, YOUNG_COMPANY); средна = ochre
(LONG_TERM, GRACE_PERIOD, NO_INDEXATION); ниска = stone (MISSING_MONEY, DATA_CONFLICT).
Съответствието код→тежест е от [red-flags.md](red-flags.md).

## Компоненти (речник)

- **Флаг бадж** — мономино код + щрихован квадрат-маркер + кратката стойност
  (`LOW_PAYMENT · 0,62% годишно`). В таблици — само щрихованият маркер с `title`.
- **Произход (`prov`)** — малък бордиран чип-линк: „НКР партида ↗", „Обявление,
  Раздел IX ↗", „JSON изглед", „CSV експорт". Стои под всяко число на детайлните
  страници и във футъра на детайла.
- **Стойност с качество** — нормализираната стойност (`259,75 лв. → 132,81 €`) винаги
  придружена от суровия низ и `value_flag` в дребен курсив
  (`източник: „259,75 лв." · parsed_from_text`). Оригиналът никога не изчезва.
- **Паспорт** — заглавният блок на детайла: вертикална партидна лента (water фон, mono),
  вид/статус ред, заглавие (Literata), страни с връзки, дясна колона с трите ключови
  числа (срок, годишно, стойност).
- **Раздели** — детайлът следва формуляра на НКР: секции с римски номера (Раздел I–XI,
  Literata) — номерацията е на първоизточника, не украса.
- **Счетоводен ред (ledger)** — mono ред с ключовите бройки + „данни към ДД.ММ.ГГГГ"
  на началната страница.
- **Плочки по вид обект** — grid с mono брой + етикет, по таксономията от
  [core-scope.md](core-scope.md).

## Достъпност и поведение

- WCAG 2.1 AA: контраст ≥ 4.5:1 за текст (палитрата е проверена), видим фокус
  (`outline` във `--water-br`), таблиците с истински `<th scope>`.
- Клавиатура: `/` фокусира търсенето; всичко е достъпно с Tab.
- `prefers-reduced-motion` се уважава; анимациите са минимални поначало (регистърът
  не мига).
- Отзивчивост до 360px: таблиците се свиват до карти-редове или скролват в собствен
  контейнер; страницата никога не скролва хоризонтално.
- Светла тема за v1 (хартиеният език е част от идентичността); тъмна тема — кандидат
  за v2 през токените.
