# ADR-0002 — Двустепенно извличане: Python bootstrap + Worker refresh

- **Статус:** Прието
- **Дата:** 2026-07-10
- **Обхват:** ETL

## Контекст

И НКР, и data.egov.bg блокират datacenter IP-та (проверено: TLS reset / 403 на ниво Apache),
т.е. нито CI, нито Cloudflare Worker могат да ги достъпят директно. Първоначалното извличане
(1434 партиди × няколко документа + ~100 набора от data.egov.bg) е еднократна тежка операция;
опресняването е леко (НКР експортът е една заявка от ~500 KB, diff-ва се).

## Решение

Две степени: (A) **bootstrap** — Python инструменти в `tools/harvest/`, пускани ръчно от
българско IP, качват суровината в R2 с датиран префикс; (B) **инкрементален refresh** —
`koncesii-etl` Worker по cron тегли само делтата през BG изходна точка (self-hosted runner
или residential proxy — изборът ще е отделен ADR при фаза 5) и обновява D1. Парсерите
съществуват двукратно: Python (bootstrap) и TS в `packages/ingest` (refresh) — TS версията е
каноничната и е покрита с тестове върху фикстури от реални страници.

## Последствия

- Плюс: bootstrap-ът не зависи от инфраструктурни решения; суровината в R2 прави всяка
  версия възпроизводима; Worker-ът остава лек.
- Минус: временно двоен парсинг код (Python/TS) — приемаме го, докато TS покрие всичко, после
  Python остава само като harvester без парсинг.
- Сурови данни не влизат в git (само фикстури за тестове, минимални и анонимизирани по
  структура).
